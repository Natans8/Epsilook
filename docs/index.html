Epsilook

Find spells by how they look and sound — a companion for Epsilon WoW

Help

Searching

  • Plain words search names, models, sounds, animations and visual effects at once; a plain number also matches a spell ID.
  • Words match any part of the text, in any order — impact fireball finds cfx_mage_fireball_impact.m2.
  • Quotes make an exact phrase: "fire bolt" matches those words together, in that order. Quotes also keep a model: inside them from becoming a tag.

Field tags

  • Type a prefix like model: (or click a field button) to search one field only. Everything you type then belongs to the tag — spaces included — until you press Enter or Tab.
  • Fields: name: model: sound: anim: fx: mech: id:. sound: and anim: also take an exact SoundKit / AnimKit ID.
  • Every tag must match (AND), and words inside one tag must match the same file or name. Several id: tags are the exception — they match any of them.
  • by spell name
  • by what it does
  • an exact SoundKit ID
  • Epsilon commands paste straight in

Effects — fx:

Visuals not tied to a model file. Each kind is searchable by its own word, plus a texture, color word, hex code or percentage.

  • red beams and chains
  • that exact tint, any effect
  • arcane materialize
  • red rim-light
  • half-faded
  • also camo, ghost, tint, desaturate
  • full-screen tints
  • forms
  • transform auras
  • summons
  • spells that make you a rideable vehicle
  • invisibility spells — reveals them; click a pill to jump between the two
  • invisibility nothing can detect

Category words

The grouped words shown as headings in the Models and Animations columns. They autocomplete inside a tag, and combine with file words in the same tag.

  • has a projectile model
  • whose projectile is an arrow
  • also trail, barrage
  • stand/walk overrides
  • just the walk ones
  • vehicle seat animations

Attachment points

Where on the model something plays. Write the attach keyword then the point, inside model: and fx: — a partial name finds them all. Missiles and beams travel between two, shown as SpellRightHand → Chest; the same model at two points shows as two pills.

  • either hand, and the spell-hand slots
  • also attach base, attach head
  • beams touching the chest
  • projectiles that fly to the head

Counting

A tag holding only a comparison counts that column's entries. The operator is required — a bare model:2 still searches file names for "2".

  • more than four models
  • exactly three animations
  • two or more sounds
  • vehicles with 3+ seats

Target-type icons

  • Who a model, sound, animation or effect plays on. A row that plays on two types shows two icons: the caster, the target, the target but never the caster, where the spell lands.
  • They search like category words in any marked column, and combine with other words in the same tag.
  • Unmarked rows simply have no target type in the game data — projectile models and their sounds come from missile sets, which carry none.
  • fire model on the caster
  • sounds played on the target
  • plays on caster and target

Excluding

  • A - prefix hides matches instead — -mech:school_damage hides every spell with that mechanic.
  • Four ways: type -model:, click the − half of a field button, click an existing tag's field label to flip it, or type - at the very start of a tag you're editing.

Editing the search bar

  • Enter, Tab or → close the tag you're typing, landing in the gap right after it; ← at the start of a tag closes it into the gap right before it instead.
  • Sitting in a gap (nothing being typed) with an empty input: ←/→ dive into editing the neighboring tag — even before the first one or after the last — so ← then typing inserts text or a new tag right there. Backspace does the same for the tag just before your cursor.
  • Click a tag's text to edit it, its × to remove it. Plain words are edited like normal text.
  • Select across tags and text with the mouse, Ctrl+A, or Shift+←/→ at the text's edge. Ctrl+C copies the selection as query text — model:book note anim:read — ready to paste back into any search bar; typing, Backspace or pasting replaces it.
  • Ctrl+Z undoes any bar edit — tags added, removed or replaced included; Ctrl+Y (or Ctrl+Shift+Z) redoes.

In the results

  • Click any tag to find all spells that use it; Shift-click excludes them instead.
  • ⧉ copies an ID; .lo, /, .mod and the Commands buttons copy ready-to-paste Epsilon commands — hover to preview. Shift-click on those wraps the copy in `backticks`.
  • ▶ on a sound file plays it (streamed from Wowhead); click again to stop.
  • Hovering a texture tag shows the texture; hovering a color swatch shows a large patch of the color with its hex and hue word.

Columns, filters, sharing

  • Hiding a column (toolbar) also excludes it from plain-word search and exports.
  • The Filter buttons narrow the current results — click one to cycle any → only spells with that (✓) → only spells without it (✕); e.g. set Sounds to ✕ to find silent spells. Filters travel with shared URLs. Sorting cycles ▲, ▼, back to relevance. The multi-value columns (Models, Sounds, Animations, Effects, Mechanics) sort by number of entries — the first click puts the most extreme spells on top.
  • The URL always holds your search — click 🔗 Share (next to Export) to copy it, or copy it directly.
  • Add &export=json or &export=csv to any shared URL and opening it downloads the results directly.
